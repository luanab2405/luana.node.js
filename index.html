O que é Node.js?



Node.js é uma tecnologia assíncrona que trabalha em uma única thread de execução. Ele não é um framework Javascript, está mais para uma plataforma de aplicação, na qual você escreve seus programas com Javascript que serão compilados, otimizados e interpretados pela máquina virtual V8.

Além disso, Node.js não é uma linguagem de programação, ele utiliza JavaScript e permite utiliza JavaScript full-stack. Ele também é muito leve e é multiplataforma, permitindo que você consiga rodar seus projetos em servidores abertos e com o SO que quiser, diminuindo bastante seu custo de hardware e software.

Para quê serve Node.js?



Node.js serve para fazer APIs. Esse talvez seja o principal uso da tecnologia, uma vez que por default ela apenas sabe processar requisições.

Node.js serve para fazer backend de jogos, IoT e apps de mensagens. Seu modelo não bloqueante de tratar as requisições o torna excelente para essa tarefa consumindo pouquíssimo hardware.

Node.js serve para fazer aplicações de tempo real. Usando algumas extensões de web socket com Socket.io, Comet.io, etc é possível criar aplicações de tempo real facilmente sem onerar demais o seu servidor como acontecia antigamente com Java RMI, Microsoft WCF, etc.

Quais as vantagens do Node.js?



  • Node.js usa Javascript. Javascript tem algumas décadas de existência e milhões de programadores ao redor do mundo.
  • Node.js permite Javascript full-stack. Uma grande reclamação de muitos programadores web é ter de trabalhar com linguagens diferentes no front-end e no back-end.
  • Node.js é muito leve e é multiplataforma. Isso permite que você consiga rodar seus projetos em servidores abertos e com o SO que quiser, diminuindo bastante seu custo de hardware e software

Quais as desvantagens do Node.js?



  • Node.js usa Javascript. Para quem gosta de linguagens estritas como Java e C#, isso incomoda bastante.
  • Node.js é recente (2009). Isso é uma desvantagem em relação à linguagens mais maduras como a JVM (1995) e Python (1991).
  • Node.js é assíncrono. Isso complica bastante dependendo da complexidade de cada uma das suas requisições, uma vez que o uso demasiado de callbacks pode gerar o chamado callback hell.
